# Project Infrastructure & Architecture Analysis
**Date:** 2026-04-09
**Status:** Internal Review

## 1. Identified Inefficiencies
*   **The "Execution Gap":** The Python CLI currenty acts as a "Directive Dispatcher" rather than a "Worker." It prints instructions for the Agent to see (double-turn) instead of performing the file I/O itself.
*   **Manifest Bloat:** `manifest.workspace.json` (~25KB) includes "noise" directories (logs, tmp, cache) that consume excessive tokens during every sync/update.
*   **Redundant State Reading:** Large YAML state files are loaded in their entirety for every command, increasing overhead as academy progress grows.
*   **Synchronous Bottlenecks:** The `sync` process is a "heavy" all-or-nothing operation, which is inefficient for minor updates to single notes.

## 2. Context Window Optimization
*   **Selective Ingestion:** Implement filtering in `SyncManager` to ignore `.venv`, `.pytest_cache`, and `evidence/logs`.
*   **Compact Data Structures:** Transition to whitespace-minimized JSON for agent-facing manifests.
*   **Differential State:** Providing only relevant "Recent History" (last 3 questions) to the agent during active sessions.
*   **Direct-to-Disk Logic:** Moving the formatting of YML/Markdown files from the Agent to the Python scripts to reduce instructional overhead.

## 3. Intake Content Strategy
*   **Preprocessing Pipeline:** Strip UI metadata (Page X/Y, Author tags) and boilerplate before presenting content to the agent.
*   **Key-Value Extraction:** Shift from paragraph-based storage to high-density bullet points/YAML keys.
*   **Knowledge De-duplication:** Check existing notes for overlap before creating new `academy/tmp/` files.

## 4. SME & Skill Integration
*   **Custom Skills (`.gemini/skills/`):** Create a `testio-expert` skill to encapsulate "AI-Proofing" and "Reproduction Standards" without bloating the main session history.
*   **MCP (Model Context Protocol):** Integrate a local database of past bug reports to enable the agent to analyze historical rejection patterns and offer personalized corrections.
*   **Sub-Agent Delegation:** Use specialized agents (`codebase_investigator`, `generalist`) for distinct phases of the Research -> Strategy -> Execution lifecycle.
