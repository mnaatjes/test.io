# Functional Bugs: Assessment and Severity Reference Notes

**Last Updated:** 2026-03-29

## Definition
### Core concept
Bugs related to the functionality of software (action-response failure).

### Examples
* Button doesn't submit
* Search doesn't react
* Filter logic returns wrong data (e.g., price range filter showing wrong prices)
* App crashes

### Identification strategy
* Differentiate between broken features and intentional design.
* Test features in isolation and in combination.
* Provide evidence (e.g., a field is validated in some cases but not others).
* If a visual/content issue hinders functionality, report it as Functional.
* Consistent behavior in different scenarios is likely 'Intended' (Usability Suggestion).


## Severity assessment factors
### Impact
How relevant is the functionality to the whole product?

### Extent
How many users, products, or items are affected?

### Workarounds
Is there an intuitive/easy alternative route? (Showstopper = No workaround for main functionality).

### Loss of sales
Secondary factor; potential financial impact on the customer.

### Comparison
Check previously approved bugs in the same test for consistency.


## Severity levels
### Low
* Minimal impact on product usage.
* General usage is not affected.
* Few users/items concerned.
* Broken feature has an easy workaround.

### High
* Serious impact, but main functionality remains intact.
* Large number of users/items concerned.
* Non-trivial functionality is broken with no workaround.
* Important functionality is broken but a workaround exists (not a showstopper).

### Critical
* Prevents core functionality of the app/site.
* Showstopper in a main process (e.g., checkout).
* Potential and notable loss of sales.


## Special categories
### Edge cases
* **definition:** Bugs occurring via unusual/uncommon sets of actions.
* **status:** Forwarded as 'Low' if relevant; rejected if irrelevant.
* **examples:** ['Minimizing app immediately after clicking', 'Rapidly opening/closing menus']

### Forced bugs
* **definition:** Nontypical behavior not reflecting normal user patterns.
* **status:** Generally OUT OF SCOPE/Rejected.
* **examples:** ['Tapping multiple elements simultaneously.', 'Random/Fast-paced button clicking.', 'Non-typical window resizing/zooming.', 'Full RAM/Internal memory issues.', 'Using unofficial/beta/modified OS versions.']
