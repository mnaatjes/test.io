# Bug Report Attachments Reference Notes

**Last Updated:** 2026-03-29

## Core policy
### Requirement
Each bug must have at least one attachment as evidence.

### Purpose
Proof of occurrence on specific device/OS/browser; does NOT replace written info.


## Selection rules
### Screencast required
* Functional bugs (always required to show action/process).
* Whenever an action triggers a bug.
* To illustrate a process (static images cannot show root cause).

### Screenshot sufficient
* Static bugs (Visual or Content issues).
* Static GUI problems.


## General requirements
### Freshness
New attachments required for every report/reproduction; NO copying.

### Language
All relevant info (date, time, system info, errors) must be in English (or German if specified).

### Device sync
Report only ONE device; upload only one attachment for it.

### Privacy
* Do not show other Test IO customers' data (tab names, emails).
* Do not show personal info (private photos, bad word autocorrect).

### Technical
* URL field must be visible for website tests.
* Resolution must be high enough to read text.
* Record the WHOLE screen.
* Crash logs are MANDATORY for app crashes (must match video timing).


## Date time rules
### Visibility
Current date and time must be visible.

### Mobile specific
If using a screenshot on mobile, a second screenshot showing date/time is required (battery/time must match).

### Format
24-hour clock preferred; 12-hour must use AM/PM.

### Location tips
* **windows:** Taskbar or calendar.
* **mac:** Calendar icon or menu bar.
* **mobile:** Swipe down notification center at start of recording.


## Formatting
### Screenshots
* JPG
* PNG
* Highlight the bug

### Screencasts
* MP4
* Max 25MB
* Max 60s for Bug Reports
* Max 15s for Reproductions


## Streaming devices
### Requirements
* Record WHOLE TV screen.
* TV Remote must be visible and clear.
* No dark ambient light.
* No background noise (conversations, TV, pets).
