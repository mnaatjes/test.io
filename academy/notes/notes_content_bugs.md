# Content Bugs Reference Notes

**Last Updated:** 2026-03-29

## Definition
### Core concept
Bugs relating to the actual content of websites or apps (text, labels, pictures, videos, icons, links, data).

### Typical examples
* Broken links or images (404s)
* Defective redirections (general)
* Missing text (e.g., empty tooltips or content areas)
* Missing content (e.g., inconsistency in icon tooltips)
* Missing translations (e.g., French labels on an English site)
* Missing products in search results (when search function works)
* Missing data


## Repetition rule
### Policy
Submit ONLY ONCE.

### Description
If a content problem occurs repetitively, even with different URLs/images, it must be reported in a single report.

### Requirement
The report should state that other instances are also concerned.

### Rejection reason
Individual reports for every occurrence will be rejected.

### Examples
* Multiple broken product pictures.
* Multiple 404 download links for manuals.
* Missing translations across several labels/snippets.


## Functional upgrade criteria
### Upgrade to functional
* If a content bug prevents a functionality.
* If a link or functional element is BROKEN (404) or MISSING ENTIRELY (empty space) in a functional component (navigation menu, header, footer, breadcrumbs).

### Stay as content
* Missing or incorrect translations (even in navigation menus).
* If the functionality can be reached intuitively via a different path.

### Severity note
Content bugs in navigational components are typically 'Low' functional bugs.


## Documentation
### Requirement
Must contain screenshots.

### Reference
See Bug Report Requirements for more details.


## Out of scope
* Spelling problems (typos).
* Dummy texts/missing content on staging environments (often intentional).
