# Visual Bugs Reference Notes

**Last Updated:** 2026-03-29

## Definition
### Core concept
Bugs relating to the graphical user interfaces (GUI) of websites or apps.

### Typical examples
* Layout framework problems (e.g., misaligned text/elements)
* Responsive Design problems (e.g., element visible on one device but not another)
* Text/elements unintentionally overlapping
* Text/elements cut off
* Anything fixable in HTML or CSS files


## Repetition rule
### Policy
Submit ONLY ONCE.

### Description
If a visual problem occurs repetitively across different URLs or pages, it must be reported in a single report.

### Requirement
The report should state that other instances are also concerned.

### Rejection reason
Individual reports for every occurrence will be rejected.

### Examples
* Multiple elements showing the same graphical scaling problem.
* Input fields across the site too small for default text.
* Multiple teasers overlapping other elements.


## Functional upgrade criteria
### Upgrade to functional
* If a visual bug prevents a functionality.

### Stay as visual
* If the functionality can be reached intuitively and easily via a different path or option.


## Documentation
### Requirement
Must contain screenshots.

### Optional
Screencast required if an action causes the visual problem.

### Reference
See Bug Report Requirements for more details.


## Responsive design insights
### Common issues
* Inconsistent scaling between different pages.
* Text becoming illegible on specific screen sizes.
* Navigation elements/targets becoming too small to tap.
* Unintended horizontal scrolling.
