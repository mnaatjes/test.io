# Visual Bugs Notes

## Definition
Visual bugs relate to the graphical user interfaces (GUI) of websites or apps. They are anything that can be fixed in HTML or CSS files.

## Typical Examples
- Layout framework problems (e.g., misaligned text/elements).
- Responsive Design problems (e.g., element visible on one mobile device but not another).
- Text/elements unintentionally overlapping.
- Text/elements cut off.

## Repetitive Problems Rule
- **Policy:** Submit ONLY ONCE.
- If a visual problem occurs repetitively (different URLs, links, or pages), it must be reported in a single report.
- **Requirement:** State in the report that other instances are also concerned.
- **Rejection:** Individual reports for every occurrence will be rejected.

## Functional Upgrade Criteria
- **Upgrade:** Report as functional if the visual bug PREVENTS functionality.
- **Exception:** It remains a visual problem if the functionality can be reached intuitively and easily via a different path or option.

## Documentation
- **Screenshots:** Mandatory for all visual reports.
- **Screencasts:** Required if an action causes the visual problem.

## Responsive Design Identification
- Look for inconsistent scaling between pages.
- Check for illegible text on specific screen sizes.
- Notice if navigation elements become too small to tap.
- Watch for unintended horizontal scrolling.

## Tools
- **Google Chrome Lighthouse:** Evaluates mobile usability factors (font size, tap targets, viewport configuration).
