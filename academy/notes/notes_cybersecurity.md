# Cybersecurity Course Notes

## Overview
Learn core concepts in Cybersecurity, identify threats, protect devices for testing, mitigate risks, and report suspicious activity.

## 1. Social Engineering Techniques
- **Definition:** Non-technical intrusion relying on human interaction/tricking people into bypassing security.
- **Common Techniques:**
    - **Simple Direct Request:** Asking for data directly.
    - **Dumpster Diving:** Searching trash for identity info.
    - **Raiding Mailboxes:** Stealing from postal mail.
    - **Phishing:** Counterfeit emails/sites to steal credentials.
    - **Impersonation:** Pretending to be authority figures (IT, managers).
    - **Surfing Company Websites:** Gathering public intel (employee info, charts).
- **Intruder's Toolkit:**
    - **Vulnerability Scanning:** "Pre-attack probe" for weaknesses.
    - **Password Cracker:** Automated guessing/dictionary attacks.
    - **Network Spoofing:** Impersonating sign-on to collect passwords.
    - **Viruses/Worms:** Malicious code that replicates.
    - **Logic Bombs:** Code triggered by specific conditions.
    - **Ransomware:** Encrypts files for ransom (Bitcoin).
    - **DoS Attacks:** Overloading resources to make them unavailable.

## 2. What Is Phishing?
- **Signs of a Phishing Email:**
    1. Unfamiliar/illegitimate address.
    2. Sense of urgency ("Urgent!", "Action Required!").
    3. Generic greeting ("Dear Customer").
    4. Spelling/grammar mistakes.
    5. Uncommon requests (wire money, sensitive info).
    6. Suspicious links (hover to verify).
    7. Suspicious attachments (.exe, unknown files).
- **DO:** Report via "Report Phish" button in Outlook.
- **DON'T:** Click links, open attachments from unknown senders, or share credentials/ID info.

## 3. Data Privacy
- **Threats:**
    - **External:** Hackers, malware, ransomware, DDoS.
    - **Internal:** Human error, intentional misuse, production data in test scripts.
- **Data Categories:** Personal (PII), Customer, Project, Company.
- **Confidentiality Classes:** Public, Confidential, Strictly Confidential.
- **PII Levels:**
    - **Confidential PII:** Name, email, address, location, photo.
    - **Strictly Confidential PII:** Race, politics, religion, biometric, health, financial, credentials, Gov ID.
- **Special Categories:**
    - **PHI:** Personal Health Information.
    - **PCI:** Payment Card Information (Test IO is PCI DSS compliant).
    - **Intellectual Property:** Financial, Legal, Source Code, Marketing.

## 4. Device Safety for Testing
- **Data Leakage Prevention:**
    - Keep browser updated; do not save passwords in browser.
    - Use strong, unique passwords + 2FA.
    - Use WPA3 Wi-Fi; use VPN on public Wi-Fi.
    - Enable device encryption and screen locks.
    - Regular backups (verify integrity).
- **Virus Protection:**
    - Reputable antivirus + regular scans.
    - Keep OS and apps updated.
- **Spam/Email Safety:**
    - Enable spam filters.
    - Avoid "unsubscribe" links in suspicious mail.
    - Verify legitimacy before acting.

## 5. DOs and DON'Ts
- **✅ DO:** Set strong passwords, lock devices, keep software updated, be cautious online.
- **❌ DON'T:** Share passwords, leave devices unattended, use outdated software, allow others access.
- **⛔ Prohibited:** P2P/Torrents (unauthorized), copyright sharing, pornography, hacking/cracking tools, anonymizers, crypto mining, non-standard encryption.
