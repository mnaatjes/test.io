# Bug Confirmations (Fix & Report) Notes

## Overview
Bug Fix and Bug Report Confirmations are individual tasks used to verify if a bug has been fixed or to provide extra information/verification outside of a regular test cycle.

## Eligibility
- Pass the "Bug Fix and Bug Report Confirmation" course.
- Submit over **50 bugs**.
- Achieve **Silver level** or higher.

## Dashboard Location
- Found under the **Bug Confirmations** section at the bottom of the Tester Dashboard.

## Workflow & Reservation
- **Timer:** Clicking "Begin" reserves the task for **30 minutes**.
- **Cancellation:** Canceling unlocks the task for other testers.

## Execution Options
### Bug Fix Confirmation
- **Bug is fixed:** If the issue is no longer present.
- **Bug is not fixed:** If the bug still exists.
- **Can't proceed:** If blocked by technical issues.

### Bug Report Confirmation
- **Bug exists:** If reproduced as described.
- **Bug doesn't exist:** If cannot reproduce as described.
- **Can't proceed:** If blocked by technical issues.

## Comments
- **Mandatory for:** "Bug is not fixed", "Bug doesn't exist", or "Can't proceed".
- **Requirements:** Be precise, objective, and focus on the user's perspective. Avoid technical jargon.

## Attachments
- **Screencast:** Required if action is needed to show the result.
- **Screenshot:** Allowed for UI-only issues.
- **Special Requirements:** Customers may request:
  - **Internet Speed:** Must show `fast.com` at the start and end of the screencast.
  - **Console/Crash Logs.**
  - **Screen Size / Browser Version.**
  - **External Camera.**

## Submission
- **Finality:** Cannot be edited after clicking "Submit".
- **Professionalism:** Feedback goes directly to the customer.
