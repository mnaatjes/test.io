# Quickly learn how to join the first specialized testing campaign Reference Notes

**Last Updated:** 2026-03-29

## Campaign onboarding
### Process
* Click referrer link.
* Follow short sign-up process.
* Add Devices and Languages to your profile via the Dashboard.

### Benefits
* Skip regular Onboarding and courses.
* Automatically approved for all tasks.
* Immediate access to running product tests (if profile matches requirements).

### Warnings
System displays red/warning bars if missing required devices or languages for a specific campaign.


## Test participation
### Accepting tests
* Click the test cycle card on the dashboard.
* Read and perfectly understand cycle instructions.
* Check 'I've read all instructions' and click 'Yes, I will test!'.

### Test sessions
* **purpose:** Required to submit bugs, reproductions, User Stories, or Test Cases.
* **mechanism:** Red 'Start Test Session' button at the top of the Test Overview page.

## Bug submission
### Button
Located in the bottom left corner.

### Pre requisites
Verify bug is not in 'Known Bugs' or already submitted by another tester.


## Post campaign transition
### Path
After a campaign ends, transition to the regular Community.

### Requirements
Pass the 'Exploratory Course' (Onboarding) to become an approved tester.

### Benefits
Unlock all regular test cycles, Bug Fix Confirmations, and Bug Report Confirmations.
